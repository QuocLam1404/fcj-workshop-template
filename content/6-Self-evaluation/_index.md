## Strengths
- **Fast adoption of IaC:** Grasped how to declare and manage cloud resources programmatically (AWS SAM CLI) instead of manually clicking through the AWS Console.
- **Technical documentation research skills:** Proficient in extracting key information from official AWS Docs (AWS Developer Guides) to independently learn and correctly apply IAM security policies and VPC configurations.
- **Persistence in troubleshooting:** Never discouraged by long AWS error traces; built the habit of inspecting CloudWatch Logs to pinpoint root causes.

## Areas for Improvement
- **Network Security:** Need a deeper understanding of Security Groups and Network ACLs in advanced multi-tier VPC topologies.
- **Cost Optimization:** Lack hands-on experience monitoring and minimizing billing for unused or over-provisioned cloud resources.
- **Professional CI/CD Pipelines:** Need to learn automated CI/CD tools rather than relying on manual deployment from my local terminal.

## Goals for Future Development
1. Pass the **AWS Certified Solutions Architect - Associate (SAA)** exam within the next 3 months.
2. Learn **Terraform** to expand my skills in Multi-Cloud Infrastructure as Code.
3. Explore containerization technologies (Docker, ECS/EKS) to diversify my infrastructure deployment options.
